数据挖掘的意义

数据挖掘的意义就在于信息时代的到来，导致大量数据的产生，但这些大量的数据却无法转化为有效的信息，或者成为知识。因此需要有效的工具用于分析这些大量的数据。数据挖掘的意义就是找到强有效的工具和方法来分析大量数据，并且在其中找到有意义的信息。

数据挖掘的定义

Data mining is the process of discovering interesting patterns and knowledge from large amounts of data.

数据挖掘就是从大量数据中找到有效的模式以及知识的方法。

数据挖掘的基本步骤

  1. 数据清理
  2. 数据整合
  3. 数据筛选
  4. 数据转换
  5. 数据挖掘
  6. 模式评估
  7. 知识展现

可以被挖掘的数据

  1. 存储在关系数据库中的数据
  2. 数据仓库中的数据
  3. 事务数据
  4. 其他各种数据，如：时间相关的交易数据等。

可以被数据挖掘发现的模式

Class/Concept Description 类型/概念的描述

Data characterization 数据特征 一组特定的数据的特征或者是特性

Data discrimination 数据区分 把一组数据的一般特征与另外一组或多组的数据的特征进行比较，找出其中的区别

Frequent Pattern: Associasion and Correlation 频繁模式：关联与相关

频繁模式就是指在一批数据中经常出现的一些数据模式，例如频繁物品集合、频繁序列、频繁结构。频繁物品集合 就是指那些在大量的事务数据中通常会出现在一起的物品，例如牛奶和面包，通常都会在商店中被一起购买。频繁序列 就是指在数据中，常常以相似的顺序出现的，例如，通常人们会先买一个笔记本电脑，然后再买一台数码相机，之后会再买一张SD卡。频繁结构 就是指那些频繁由相似的频繁序列和频繁物品集合组合成的结构。

通过对数据中的频繁模式进行数据挖掘，我们就可以得到数据中数据的关联性与相关性。

Classification and Regression for Predictive Analysis 用于预测分析的分类与回归

分类分析 通常用于在数据中找到一个模型来描述一个特定的物品类型的特征。可以事先用一个训练数据集合来得到这个模型。分类分析的方法有很多：决策树、规则引擎、神经网络、平凡贝叶斯、k-NN以及支持向量机(SVM)。

回归分析 是一种统计学上的方法。上面的分类分析更多是基于离散的数据进行的，而回归分析则是多用于连续的数据，用于预测发展的趋势，并且能得到相关的数字。

Cluster Analysis 聚类分析

聚类分析不同于分类分析，分类分析通常是基于一些已经被标记的数据而进行分析的。但是数据往往在一开始是没有任何分类标记的，这时，聚类分析就可以发挥作用。聚类分析的原则可以简单描述为：最大化同类中的相似度，而最小化不同类的相似度。
